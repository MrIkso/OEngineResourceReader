OEngine Resource Reader v%version%

Основні можливості:
- Перегляд графічних файлів (текстур).
- Перегляд шрифтів.
- Перегляд та редагування текстових файлів.
- Заміна текстур.
- Заміна шрифтів.

Робота з файлами:
- Відкриття окремого текстового документа: перетягніть файл у вікно програми або скористайтеся меню File -> Load Text.
- Відкриття окремої текстури: перетягніть файл у вікно програми або скористайтеся меню File -> Open Texture File.
- Відкриття окремого файлу шрифту: перетягніть файл у вікно програми або скористайтеся меню File -> Load Font.
- Відкриття всієї папки: для зручної навігації по проєкту перетягніть папку (зазвичай це _Cooking у теці з ресурсами гри) у вікно програми або скористайтеся меню File -> Open Resource Directory.
- Експорт текстури у готовий до редагування графічний файл .dds: File -> Export To Texture File.
- Заміна поточної текстури на інший графічний файл .dds: File -> Replace Texture. Після цього необхідно зберегти зміни через File -> Save Modified Texture.
- Експорт конфігурації шрифту для подальшого створення нового шрифту в BMFont: File -> Export Font Configuration to BMFont.
- Генерація нового шрифту на основі файлу .fnt, створеного у програмі BMFont: File -> Generate new font from .fnt.

Інтерфейс:
- Папки:
    - Навігація по дереву файлів здійснюється мишею або стрілками на клавіатурі. Файл відкривається кліком миші або клавішею Enter.
    - Контекстне меню для елемента в дереві викликається правою кнопкою миші (ПКМ).
    - Під деревом файлів розташований фільтр, де назви або розширення файлів можна вводити через кому (наприклад: *.dxt, Text).
    - У контекстному меню є можливість відкрити у Провіднику Windows або скопіювати шлях до в буфер обміну.

- Файли:
    - Файли різних типів (текст, текстура, шрифт) відкриваються у власних вкладках. Угорі вікна програми відображається назва файлу з поточної вибраної вкладки.
    - Text:
        - Текстовий файл можна редагувати безпосередньо в програмі. Угорі відображається кількість рядків у стовпці "Original Text". Переклад записується у стовпець "Translation".
        - Кнопка Save зберігає зміни у файл, місце для якого вказується в діалоговому вікні.
        - Кнопка Export експортує поточний вигляд таблиці (а не початковий вміст файлу) у формат .json. Цей файл можна завантажувати у сторонні сервіси, наприклад Crowdin чи SmartCat.
        - Кнопка Import імпортує раніше експортований .json назад до програми, заносячи дані до стовпця "Translation", якщо вони відрізняються від даних у "Original Text".
        - File Version — технічний параметр, що не має значення для кінцевого користувача.
        - Поле пошуку за словами працює для обох стовпців.

    - Texture:
        - Підтримується масштабування коліщатком миші з натиснутою клавішею Ctrl та жестами на тачпаді. Клік ЛКМ збільшує зображення, Ctrl + клік ЛКМ — зменшує.
        - Клавіша B (або W) перемикає фон вікна перегляду між чорним та білим.
        - Texture Format — технічна інформація, що знадобиться для створення та заміни текстури у сторонній програмі. Нова текстура повинна мати такий самий формат.

    - Font:
        - Є вікно попереднього перегляду, інформація про шрифт, а також дані про гліф, на який наведено курсор миші.
        - Live Preview Text — поле, де можна ввести будь-які символи, щоб побачити, як вони відображатимуться обраним шрифтом, та перевірити, які з них підтримуються.

Робочий процес:
- Заміна тексту:
    1. Редагуйте текст безпосередньо в програмі (зберігаються лише рядки, де поле "Translation" не порожнє; решта залишається без змін).
    2. Або експортуйте файл у .json, відредагуйте у сторонньому сервісі, експортуйте звідти у такий самий формат .json та імпортуйте назад до програми. Після цього збережіть файл.
- Заміна текстури:
    1. Відкрийте файл текстури в програмі.
    2. Експортуйте її як зображення .dds.
    3. Відредагуйте у сторонньому редакторі (наприклад, GIMP).
    4. Замініть зображення в програмі (File -> Replace Texture) та збережіть його (File -> Save Modified Texture).
- Заміна шрифтів:
    1. Відкрийте файл шрифту (не текстуру).
    2. Експортуйте файл конфігурації шрифту.
    3. Створіть новий шрифт та його текстуру за допомогою BMFont (https://www.angelcode.com/products/bmfont/).
       Важливо: експортований з BMFont файл опису шрифту має бути в текстовому форматі (.fnt), а сам файл .fnt та файл текстури шрифту повинні знаходитись в одній папці.
    4. Згенеруйте новий шрифт на основі створеного файлу .fnt через File -> Generate new font from .fnt. Текстура шрифту автоматично заміниться на нову й буде збережена.
       Важливо! При збереженні шрифту його потрібно зберегти з такою самою назвою, як в оригінального файлу, щоб гра змогла його розпізнати, а текстура шрифту — замінитися.
- Заміна шрифтів для старої версії:
    1. Відкрийте файл шрифту.
    2. Експортуйте конфігурацію шрифту для BMFont.
    3. Створіть новий шрифт у BMFont та збережіть його у текстовому форматі (.fnt).
    4. Скопіюйте текст нового .fnt файлу та замініть в оригінальному .fnt файлі шрифту. 
       Пам'ятайте, що назва файлу шрифту має залишатися незмінною та назва текстури шрифту яка є в .fnt файлі також повинна відповідати оригінальній, щоб гра змогла її розпізнати.
    5. Замініть текстуру шрифта на нову текстуру, створену у BMFont.

- Перегляд шрифтів:
    1. Відкрийте файл шрифту.
    2. Перегляньте гліфи, їх розміри та інші параметри.
    3. Використовуйте поле Live Preview Text для перевірки підтримки символів.
       Також, в блоці інформації про шрифт можна побачити яку саме текстуру використовує шрифт, її розмір.
